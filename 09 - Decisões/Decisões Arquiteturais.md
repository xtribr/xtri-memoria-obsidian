# Decisões Arquiteturais

## Objetivo da nota

Registrar decisões técnicas relevantes para evitar retrabalho e perda de contexto.

## Decisões vigentes

### Separação de stacks por contexto

- Projetos novos usam Next.js + React + Tailwind + Supabase.
- O site institucional legado permanece em WordPress + PHP.
- Análises de dados usam Python e R/mirt quando necessário.

### Política de dados

- Não utilizar dados fictícios em tarefas que impactem análises ou decisões.
- Toda fonte crítica deve ser rastreável.
